# 项目技术文档

## 1. 项目概述

本项目用于抓取海外短剧站点数据，统一多站点字段后按天导出 CSV，并通过 FastAPI 对外提供只读下载接口。

当前接入站点：

- NetShort
- DramaBox
- ReelShort
- ShortMax

项目核心原则：

- 站点抓取逻辑独立
- 通用能力下沉到 `src/core`
- 页面主链优先，必要时少量兜底
- 对外字段统一，不暴露不稳定指标

## 2. 技术栈

- Python 3.12+
- uv
- FastAPI
- APScheduler
- httpx
- BeautifulSoup4
- pytest

## 3. 系统架构

整体分为四层：

### 3.1 调度与批任务层

- `src/jobs/daily_crawl.py`
  - 负责单次抓取任务执行
  - 支持单站和全站执行
  - 负责成功写 CSV、失败记日志
- `src/jobs/scheduler.py`
  - 负责每天 `00:00` 定时调度
  - 时区使用 `Asia/Shanghai`

### 3.2 API 层

- `src/api/app.py`
  - 提供健康检查接口
  - 提供 CSV 下载接口
  - 不参与抓取执行

### 3.3 Core 通用能力层

- `src/core/settings.py`
  - 统一配置加载
- `src/core/http.py`
  - HTTP 请求封装、重试、超时
- `src/core/models.py`
  - 统一数据模型与 CSV 字段定义
- `src/core/normalize.py`
  - 数值标准化、日期标准化、受众类型映射、集预览截断
- `src/core/export.py`
  - CSV 原子写入
- `src/core/storage.py`
  - 导出文件与目录路径管理
- `src/core/logging.py`
  - 每日日志输出
- `src/core/base.py`
  - 爬虫基类与通用方法

### 3.4 Spider 站点层

- `src/spiders/netshort/crawler.py`
- `src/spiders/dramabox/crawler.py`
- `src/spiders/reelshort/crawler.py`
- `src/spiders/shortmax/crawler.py`

各站点只负责：

- 获取站点入口页面
- 解析原始字段
- 映射到统一 `DramaRecord`

## 4. 数据模型

### 4.1 剧级统一字段

| 字段 | 说明 |
| --- | --- |
| `site` | 站点名称 |
| `site_drama_id` | 源站剧 ID |
| `title` | 标题 |
| `summary` | 简介 |
| `cover_url` | 封面图链接 |
| `detail_url` | 详情页链接 |
| `tag_list` | 标签列表，导出时用 `|` 拼接 |
| `category_list` | 分类列表，导出时用 `|` 拼接 |
| `audience_type` | 受众类型：`male` / `female` / `mixed` / `unknown` |
| `publish_date_std` | 标准化上线日期 |
| `play_count` | 剧级播放量 |
| `collect_count` | 剧级收藏量 |
| `like_count` | 剧级点赞量 |
| `episode_count` | 集数 |
| `episodes_preview_json` | 最多前 10 集预览 |
| `metric_source` | 指标来源：`page` / `mixed` / `cover_url_path` 等 |
| `crawl_date` | 抓取日期 |
| `crawled_at` | 抓取时间 |

### 4.2 集预览字段

`episodes_preview_json` 由以下字段组成：

- `episode_id`
- `episode_no`
- `episode_type`
- `episode_url`
- `is_locked`
- `play_count`
- `collect_count`
- `like_count`

## 5. 站点抓取设计

### 5.1 NetShort

主链：

- 优先从首页抓取 `/episode/...` 或 `/full-episodes/...` 种子
- 进入详情页提取主数据
- 尝试请求 `/hotseries/...` 补发布日期

当前实现状态：

- 可抓取
- 存在部分剧 `hotseries` 返回 `404`，此时发布日期可能为空

### 5.2 DramaBox

设计链路：

- `/browse/0`
- 分类递归页 `/browse/{id}`
- 详情页 `/drama/{bookId}/{slug}`

当前实现状态：

- 代码已实现
- 当前环境请求主站即被 CloudFront 拦截，返回 `403`
- 因此生产抓取暂未打通

### 5.3 ReelShort

主链：

- `/movie-genres/all-movies`
- 详情页 `/movie/{slug}-{book_id}`
- 通过 `__NEXT_DATA__` 解析剧级信息
- 从 `online_base` 生成集级预览

当前实现状态：

- 可抓取
- 当前是四站里最稳定的一站

### 5.4 ShortMax

主链：

- `/dramas`
- 分页与分类页递归发现 `/episode/...-1`
- 从首集页直接提取主数据
- 封面 URL 路径提取发布日期

当前实现状态：

- 可抓取
- 全量抓取耗时较长，但已可以稳定导出

## 6. 导出设计

导出路径：

```text
data/exports/{site}/{site}_{YYYY-MM-DD}.csv
```

导出规则：

- 每站点每天一个 CSV
- 采用临时文件 + 替换方式原子写入
- 空结果视为失败，不覆盖旧文件
- 统一使用 UTF-8 编码

## 7. 调度设计

调度器使用 `APScheduler BlockingScheduler`。

固定任务：

- 任务 ID：`daily-drama-crawl`
- 触发时间：每天 `00:00`
- 时区：`Asia/Shanghai`
- 执行动作：调用 `run_daily_crawl(..., selected_site="all")`

## 8. 日志与排障

日志文件：

```text
logs/{YYYY-MM-DD}.log
```

每个站点会记录：

- 开始时间
- 成功 / 失败状态
- 记录数
- 输出文件路径
- 失败异常堆栈

常见问题：

### 8.1 CSV 未生成

排查顺序：

1. 看对应日期日志
2. 看站点是否返回空结果
3. 看站点是否被风控或返回非 200

### 8.2 DramaBox 无数据

当前不是解析问题，而是访问层 `403`，需要补代理、浏览器态 cookie 或真实浏览器抓取方案。

### 8.3 ShortMax 抓取时间长

全量分页较多，属于当前实现特性；如需进一步提速，可后续增加并发控制或增量种子机制。

## 9. 配置说明

| 环境变量 | 默认值 | 说明 |
| --- | --- | --- |
| `DRAMA_TIMEZONE` | `Asia/Shanghai` | 调度与日期计算时区 |
| `DRAMA_DATA_DIR` | `data` | 数据根目录 |
| `DRAMA_EXPORTS_DIR` | `data/exports` | 导出目录 |
| `DRAMA_LOGS_DIR` | `logs` | 日志目录 |
| `DRAMA_HTTP_TIMEOUT` | `20` | HTTP 超时秒数 |
| `DRAMA_HTTP_DELAY_SECONDS` | `0` | 每次请求后的附加等待 |
| `DRAMA_HTTP_MAX_RETRIES` | `2` | HTTP 重试次数 |
| `DRAMA_USER_AGENT` | 内置桌面 UA | 请求头 UA |
| `DRAMA_MAX_ITEMS` | 空 | 调试模式下限制记录数 |
| `DRAMABOX_MAX_PAGES` | `250` | DramaBox 页数上限 |
| `REELSHORT_MAX_PAGES` | `200` | ReelShort 页数上限 |
| `SHORTMAX_DRAMAS_URL` | `https://www.shorttv.live/dramas` | ShortMax 列表入口 |
