## 1. 统一口径

- 这版只保留真正要落库和对外承诺的字段，不再展开研究型字段。
- 统一原则：页面主链优先，API 只做兜底。
- 剧级必保字段：标题、简介、封面、详情链接、标签/类型、上线日期、剧级数值。
- 集级只保留真正稳定的数值字段；当前只有 ReelShort 有稳定的单集点赞。
- `热度值` 当前四站都没有稳定生产字段，先不承诺。
- `男频/女频` 当前没有四站统一的稳定原始真值字段，建议统一落一个派生字段 `audience_type`，由标签/分类映射得到。

## 3. 四站点简版口径

### NetShort

**实际保留字段**

- 剧级：`shortPlayName`、`shotIntroduce`、`shortPlayCover`、`fullEpisodeNameUrl`、`labelList`、`publish_date_unified`、`totalLikeNums`、`totalChaseNums`
- 集级：`episodeId`、`episodeNo`、`isLock`
- 不承诺：单集播放/收藏/点赞/热度；剧级播放量

**获取方式**

- 全量入口：`all-plots`
- 页面补充：`full-episodes / episode / hotseries`
- API 兜底：`/web/v4/short_play/detail_info/cascade_label`
- 日期兜底：`hotseries Release date -> marketingDetails.publishTime -> sitemap 最小 publication_date`

**真实示例**

- `title = OMG! My Ride Is a ROOMBA?`
- `shortPlayId = 2039278148570316802`
- `totalLikeNums = 2.0K`
- `totalChaseNums = 1.6K`
- `publishTime = 1776182400202`

**每日全量抓取**

1. 抓 `all-plots` 全量列表，沉淀剧索引。
2. 解析标题、简介、封面、标签、详情路径。
3. 缺字段时补抓 `detail_info`。
4. 单独补发布日期，按 `hotseries -> marketingDetails -> sitemap` 兜底。

### DramaBox

**实际保留字段**

- 剧级：`bookId`、`bookName/bookNameEn`、`introduction`、`cover_url`、`drama_url`、`type_two_names/typeTwoIds`、`publish_date_std`、`viewCount`、`followCount`
- 集级：只保留集列表和基础元数据，不保留单集互动指标
- 不承诺：整剧独立点赞；单集播放/收藏/点赞/热度

**获取方式**

- 全量入口：`/browse/0` 前 250 页 + 递归全部 `/browse/{typeTwoId}`
- 详情页：`/drama/{bookId}/{bookNameEn}`
- 文章页：`/resources/review/{articleId}_{articleTitleEn}`，按需抓
- API：不进入主链

**真实示例**

- `title = The One That Got Away`
- `bookId = 42000007205`
- `firstShelfTime = 2026-03-13 10:16:29`
- `viewCount = 23,472,153`
- `followCount = 643,957`
- `detail_url = https://www.dramabox.com/drama/42000007205/The-One-That-Got-Away`

**每日全量抓取**

1. 先抓 `/browse/0` 前 250 页。
2. 从列表页发现全部 `browse route id`，递归抓分类页补齐漏剧。
3. 对每部剧抓 `drama detail`，补简介、封面、播放/收藏、日期。
4. 仅在需要文章内容时再抓 review 页。

### ReelShort

**实际保留字段**

- 剧级：`book_id`、`book_title`、`special_desc`、`book_pic`、`movie_url`、`tag/tag_list`、`publish_date_std`、`read_count`、`collect_count`
- 集级：`chapter_id`、`serial_number`、`chapter_type`、`chapter_like_count`、`episode_url`
- 不承诺：单集播放/收藏/热度；整剧总点赞

**获取方式**

- 全量入口：`/movie-genres/all-movies`
- 详情页：`/movie/{slug}-{book_id}`
- 集页：`/episodes/{episode|trailer}-{slug}-{book_id}-{chapter_id}`
- API：仅 `/api/video/hall/info` 可做补充，不进主链

**真实示例**

- `title = Queen's Gambit`
- `book_id = 69ce415907eda6f231015eb5`
- `read_count = 316043`
- `collect_count = 14425`
- `publish_at = 1776218704 -> 2026-04-15`
- `episode chapter_id = ehtws2n7t8`，且该集有 `video_url`

**每日全量抓取**

1. 抓 `/movie-genres/all-movies` 全部分页。
2. 进入 `movie detail`，补标题、简介、封面、标签、剧级数值、发布日期。
3. 展开 `online_base` 生成集索引，并保留 `like_count`。
4. 只有需要单集简介、时长或视频地址时再抓 `episode detail`。

### ShortMax

**实际保留字段**

- 剧级：`shortPlayId`、`shortPlayName`、`summary`、`coverId`、`drama_url`、`labelList`、`classList`、`publish_at`（由 `coverId` URL 路径日期提取）、`collectNum`、`playNum`
- 集级：`episodeNo`、`coverId`
- 不承诺：整剧独立点赞；单集点赞/收藏；单集播放字段虽存在但样例恒为 `0`；单集发布日期

**获取方式**

- 全量入口：首页 -> `/dramas` -> `/dramas/{page}` -> `/dramas/{class-slug}-{classId}`
- 主数据页：`/episode/{slug}-{shortPlayId}-1`
- 详情页 fallback：`/drama/{slug}-{shortPlayId}`
- API：不进入主链；`/cmsShortPlay/queryDetail` 仅保留研究用途
- 日期规则：从主封面 `coverId` 的 URL 路径中提取 `YYYY/MM/DD`

**真实示例**

- `title = Reclaiming My Throne`
- `shortPlayId = 19481`
- `collectNum = 45668`
- `playNum = 137004`
- `publish_at = 2026-04-15`
- `publish_at_source = cover_url_path`
- `detail_url = https://www.shorttv.live/drama/reclaiming-my-throne-19481`
- 单集 `playNum` 在样例中全部为 `0`

**每日全量抓取**

1. 枚举首页、`/dramas`、分页、分类页，沉淀 drama/episode seed URL。
2. 以首集页 `/episode/...-1` 为主入口，提取剧级核心字段和完整 `episodeList`。
3. 需要补 `title / description / canonical` 时再抓 `drama detail`。
4. 不把任何前端 API 放进生产主链。

## 4. 最终交付建议

- Phase 1 统一对外承诺字段：
  - 剧级：标题、简介、封面、详情链接、标签列表/类型、上线日期、剧级播放/收藏/点赞中实际存在的字段
  - 集级：只保留站点稳定存在的字段；目前仅 ReelShort 承诺单集点赞
- 统一不要承诺：
  - 四站统一 `热度值`
  - 四站统一 `男频/女频原始真值字段`
  - NetShort / DramaBox / ShortMax 的单集互动指标
  - ShortMax 的单集播放量真实性
- 建议统一派生字段：
  - `audience_type`：由标签/分类映射
  - `publish_date_std`：统一标准化日期
  - `metric_source`：标记页面 / API / 兜底来源
