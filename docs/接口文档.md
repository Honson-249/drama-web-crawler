# 接口文档

本文档面向调用方，说明如何获取每日导出的 CSV 数据。

## 1. 基础信息

- 服务协议：HTTP
- 数据格式：`text/csv`
- 基础路径：`/api/v1`

## 2. 接口列表

### 2.1 健康检查

**请求**

```http
GET /api/v1/healthz
```

**响应示例**

```json
{
  "status": "ok"
}
```

**说明**

- 仅用于判断服务是否在线
- 不代表某个站点当天抓取一定成功

### 2.2 下载站点导出文件

**请求**

```http
GET /api/v1/exports/{site}
```

**Path 参数**

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `site` | string | 是 | 站点标识 |

支持的 `site` 值：

- `netshort`
- `dramabox`
- `reelshort`
- `shortmax`

**Query 参数**

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `date` | string | 否 | 导出日期，格式 `YYYY-MM-DD` |

**行为说明**

- 传 `date`：返回指定日期的 CSV
- 不传 `date`：返回该站点最新可用 CSV

**成功响应**

- 状态码：`200 OK`
- `Content-Type`: `text/csv`
- `Content-Disposition`: 附带下载文件名

**失败响应**

#### 站点不存在

状态码：`404`

```json
{
  "detail": "unknown site: xxx"
}
```

#### 文件不存在

状态码：`404`

```json
{
  "detail": "csv export not found"
}
```

## 3. 调用示例

### 3.1 获取最新 ReelShort 导出

```bash
curl -OJ "http://127.0.0.1:8000/api/v1/exports/reelshort"
```

### 3.2 获取指定日期 ShortMax 导出

```bash
curl -OJ "http://127.0.0.1:8000/api/v1/exports/shortmax?date=2026-04-19"
```

### 3.3 健康检查

```bash
curl "http://127.0.0.1:8000/api/v1/healthz"
```

## 4. CSV 字段说明

所有站点导出使用同一套列定义。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `site` | string | 站点展示名 |
| `site_drama_id` | string | 源站剧 ID |
| `title` | string | 标题 |
| `summary` | string/null | 简介 |
| `cover_url` | string/null | 封面链接 |
| `detail_url` | string | 详情页链接 |
| `tag_list` | string | 标签列表，使用 `|` 拼接 |
| `category_list` | string | 分类列表，使用 `|` 拼接 |
| `audience_type` | string | `male` / `female` / `mixed` / `unknown` |
| `publish_date_std` | string/null | 标准化日期 |
| `play_count` | integer/null | 剧级播放量 |
| `collect_count` | integer/null | 剧级收藏量 |
| `like_count` | integer/null | 剧级点赞量 |
| `episode_count` | integer/null | 集数 |
| `episodes_preview_json` | string | 最多前 10 集的 JSON 预览 |
| `metric_source` | string | 指标来源 |
| `crawl_date` | string | 抓取日期 |
| `crawled_at` | string | 抓取时间 |

## 5. `episodes_preview_json` 说明

该字段是一个 JSON 字符串，不是展开列。

单个元素结构：

```json
{
  "episode_id": "bbxjli84ai",
  "episode_no": 1,
  "episode_type": "episode",
  "episode_url": "https://example.com/episodes/xxx",
  "is_locked": false,
  "play_count": null,
  "collect_count": null,
  "like_count": 10801
}
```

说明：

- 最多保留 10 集
- 按集号升序
- 某些站点不提供集级互动字段，则对应值为空

## 6. 使用建议

- 外部服务拉取时优先传 `date`，避免拿到非预期最新文件
- 对 `null` 字段做好兼容，尤其是不同站点不稳定的指标
- 不要依赖 `episodes_preview_json` 固定有 10 条或固定字段非空

## 7. 已知限制

- `DramaBox` 当前在本环境下可能无法产出 CSV，因为站点访问被 CloudFront `403` 拦截
- 健康检查通过不等于四个站点当天全部有数据
- 若当天站点抓取失败，接口会返回最近一次成功文件，前提是调用时不指定 `date`
